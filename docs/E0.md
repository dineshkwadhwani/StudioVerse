# E0 — Static Website + Deployment

Status: Working markdown version for Copilot context.  
Canonical source: Formal E0 specification document.

## Objective

Build the public-facing single-page marketing website for Coaching Studio using Next.js App Router and deploy it through the existing Vercel setup.

This page is the public discovery layer for the platform. It must explain the value proposition, guide visitors to key sections, support web and mobile responsiveness, expose a persistent chatbot entry point, and be production-deployable on the agreed architecture.

## Scope

Epic E0 covers:
- Landing page
- Navigation
- Footer
- Responsive behavior
- SEO
- Hosting
- Domain behavior
- Analytics
- Routing
- Error handling

This is a public marketing experience, not the authenticated app shell.

## Prerequisites

Assume T0, T1, T2, T3, and T4 are complete enough to support:
- project setup
- frontend architecture
- Firebase/backend baseline
- deployment pipeline
- security and access-control foundation

## Product positioning

Coaching Studio should be presented as a unified growth ecosystem for the coaching industry.

Core messaging:
- One platform for coaching, assessments, programs, and transformation
- Built for coaches, coaching companies, and individuals
- Turn assessments into insight, insight into action, and action into measurable transformation
- Programs, tools, events, and AI-supported reports in one connected experience

## In scope

The page must:
- communicate the platform value proposition clearly
- support single-page anchored navigation
- support responsive layouts across desktop, tablet, and mobile
- include a persistent chatbot launcher
- be SEO-ready
- be analytics-ready
- be deployable through the existing Vercel setup
- follow the existing frontend and architecture conventions

## Out of scope

Do not build the following in E0:
- authentication flows
- protected pages
- app shell behavior
- dashboards
- admin console
- program detail pages
- tool execution flows
- event registration flows
- report generation flows
- marketplace payments
- full chatbot intelligence
- CMS or admin content authoring tools

E0 includes only the chatbot launcher and placeholder modal/panel experience, not the real assistant intelligence.

## User segments

The landing page should speak to:
- Independent coaches
- Coaching companies
- Individual professionals / coachees
- General visitors

Primary outcomes should be exploration, conversion, chatbot engagement, and eventual sign-in / registration.

## Experience principles

The site should follow these principles:
- Single-page, section-based storytelling
- Fast-loading premium SaaS visual language
- Clear value proposition above the fold
- Mobile-first responsiveness
- Sticky header and persistent chatbot launcher
- Minimal friction to content discovery
- Accessibility-first implementation
- Externalized styling and behavior logic
- No ad hoc inline implementation in page content files

## Required sections

The public page should be implemented as a single route with anchored sections in this order:
1. Header / Navigation
2. Hero
3. Platform overview
4. Benefits for coaches
5. Benefits for individuals
6. Assessment and transformation section
7. Programs section
8. Tools section
9. Events section
10. Impact counters / social proof
11. Sign In / Register CTA
12. Footer

## Navigation rules

Navigation items must be:
- About
- Programs
- Tools
- Events
- Sign In / Register

Brand/logo on the left should return to `#home`.

Anchor mapping:
- Coaching Studio → `#home`
- About → `#about`
- Programs → `#programs`
- Tools → `#tools`
- Events → `#events`
- Sign In / Register → `#auth`

Behavior:
- Sticky header
- Smooth scroll
- Active section indication
- Mobile hamburger menu
- Sign In / Register emphasized as CTA

Recommended CTA labels:

## Counters

Show four counters:
- Coaches Supported
- Professionals Benefited
- Programs Available
- Events Conducted

E0 should use static configurable seed values from a dedicated content/config file.

Recommended launch defaults:
- Coaches Supported: 100+
- Professionals Benefited: 1,000+
- Programs Available: 25+
- Events Conducted: 50+

Rules:
- Labels must remain visible even if values fail
- Counter animation is optional
- Respect reduced-motion settings

## Benefits sections

Create separate sections for:
- coaches
- individuals / professionals

Coaches section should cover:
- structured client journeys
- programs and tools in one place
- assessment-based insight
- AI-supported reports and transformation planning
- event-led engagement
- professional credibility

Individuals section should cover:
- guided self-discovery
- access to programs and assessments
- personalized growth insight
- trackable transformation journey
- events and continuous learning

Each audience section should use around 3 to 6 feature cards.

## Assessment and transformation section

This section should clearly explain:
1. User completes an assessment
2. Platform interprets results
3. Results contribute to a transformation plan
4. Programs, tools, and events support progress over time

This is a key differentiator and should connect assessments with structured growth and AI-supported reporting.

## Chatbot requirements

A persistent chatbot launcher must remain visible throughout page scroll.

Requirements:
- Fixed at bottom-right safe area
- 56 px desktop, 52 px mobile
- Accessible label
- Opens modal or side panel
- Dismissible by close icon, Escape key, backdrop click, and explicit close action
- Placeholder assistant content is acceptable in E0

Example placeholder:
- greeting line
- short assistant description
- CTA for later continuation
- optional contact fallback

## Footer requirements

Footer must include:
- copyright line
- privacy policy link
- address
- Instagram
- Phone
- WhatsApp
- LinkedIn

Recommended additions:
- email
- small brand description
- “All rights reserved”

Rules:
- Footer content should come from centralized config
- Missing optional links should be hideable safely
- Do not hardcode repeated footer data across components

## Responsive requirements

Supported breakpoints:
- Mobile portrait: 360 to 767 px
- Tablet: 768 to 1023 px
- Desktop: 1024 px and above
- Large desktop: 1440 px and above

Layout guidance:
- Desktop max width around 1200 px
- Desktop padding 32 px
- Tablet padding 24 px
- Mobile padding 16 px
- Desktop 12-column behavior
- Mobile single-column stacked layout

Portrait-first mobile behavior:
- Optimize for portrait
- On mobile landscape, show a lightweight rotate-back overlay message
- Do not assume browser orientation lock

## Modal behavior

All modals/popups must support:
- backdrop overlay
- close icon top-right
- Escape key close
- focus trap
- background scroll lock
- safe mobile width
- accessible interaction patterns

## SEO

Implement with Next.js metadata support:
- unique page title
- meta description
- canonical URL
- Open Graph title, description, URL, site name, locale, image
- Twitter/X card metadata
- robots directives
- favicon and app icons

SEO/content rules:
- one H1 only
- semantic landmarks
- clean anchor IDs and internal links
- descriptive alt text
- server-renderable primary marketing copy
- optimized images
- target strong Lighthouse scores

Core keyword themes:
- coaching platform
- coaching programs
- coaching assessments
- coaching tools
- transformation plan
- professional growth platform
- coaching events

## Hosting and domain

Hosting baseline:
- frontend deployed on Vercel
- production tied to main branch
- preview deploys enabled for pull requests
- environment separation follows dev / staging / prod setup
- static assets should use Next.js / Vercel optimization

Domain rules:
- support a canonical production domain
- HTTPS / SSL active
- standardize apex vs www
- canonical metadata must match the chosen host
- alternate host should redirect permanently
- trailing slash behavior should be consistent

## Analytics

Capture at minimum:
- page view
- scroll depth: 25%, 50%, 75%, 100%
- nav link clicks
- hero CTA clicks
- Sign In / Register CTA clicks
- chatbot launcher click
- chatbot modal open
- chatbot modal close
- footer social clicks

Recommended event names:
- landing_page_view
- landing_nav_click
- landing_cta_click
- landing_scroll_depth
- chatbot_launcher_click
- chatbot_modal_open
- chatbot_modal_close
- footer_social_click

Rules:
- use one analytics provider only for MVP
- wrap analytics behind a small helper
- avoid unnecessary PII collection on the public page

## Routing

Routing rules:
- primary page route is `/`
- use fragment-based anchored sections
- nav updates URL fragment without full reload
- direct access to anchor URLs should land correctly
- sticky header offset must be handled
- unknown public routes use normal 404 behavior

Recommended implementation structure:
- `src/app/page.tsx`
- `src/app/error.tsx`
- `src/app/not-found.tsx`
- `src/modules/marketing/`
- `src/constants/site-content.ts`
- `src/lib/analytics.ts`
- `src/lib/seo.ts`

## Error handling

E0 should handle gracefully:
- missing decorative assets
- analytics failure
- chatbot placeholder failure
- invalid external social links

Rules:
- core content must remain usable if analytics fails
- image failure must not break layout
- chatbot failure should show fallback messaging
- external links should be validated via config where possible
- follow centralized frontend error utilities and app-level route boundaries

## Technical implementation rules

Use:
- Next.js App Router
- TypeScript
- Tailwind
- existing architecture conventions

Follow these rules:
- no business logic in UI components
- no direct Firestore reads from UI
- keep config/content centralized
- server-render main page shell where possible
- use client components only for interactive behavior
- keep hydration light
- no secrets in frontend code
- no privileged backend logic in frontend
