## 1. Purpose

This document defines the business and functional requirements for the **Manage Users** capability for **Company** and **Professional** users in StudioVerse.

The purpose of this Epic is to extend user creation beyond Super Admin so that a Company can create Professionals and Individuals within its own tenant, and a Professional can create Individuals associated with that Professional.

This Epic builds on the already-implemented Super Admin user management flow, where Super Admin can create Company, Professional, and Individual users across the platform.

---

## 2. Audience

### Business Stakeholders
This document explains how StudioVerse enables controlled downstream user creation inside tenant boundaries so Companies can onboard their own users and Professionals can onboard their own Individuals.

### Product Owners / Product Management
This document defines role-based user creation scope, mandatory fields, association rules, optional coach selection, and tenant inheritance behavior.

### Engineering Teams
This document provides the functional requirements needed to implement role-aware creation forms, tenant auto-association, coach association logic, and scoped user visibility.

### QA Teams

## 3. Executive Summary

StudioVerse already supports user creation through **Super Admin**, and that functionality is working for all major user types including **Company**, **Professional**, and **Individual**.

This Epic extends Manage Users so that:
- a **Company** can create **Professionals** and **Individuals**,
- a **Professional** can create **Individuals**,
- and all created users are automatically associated correctly based on the logged-in creator’s role and tenant context.

When a Company creates a Professional or Individual, the tenant must be inherited automatically from the Company, so tenant selection must not be shown on the form. When a Company creates an Individual, a **Coach** dropdown should appear as an optional field, populated with existing Professionals of that Company. When a Professional creates an Individual, the Professional must be automatically associated to that Individual without asking the user to select a coach.

---

## 4. Business Context

StudioVerse is a multi-tenant platform in which Companies act as tenant-level entities and Professionals operate within those Companies, while Individuals participate in programs, assessments, and related journeys.

Because user onboarding often happens within the Company or coach relationship rather than centrally through Super Admin, the platform needs delegated user creation that respects role scope and tenant association automatically.

This creates a more scalable operating model while preserving control, consistency, and downstream linkage to cohorts, assigned activities, and other platform modules.

---

## 5. Problem Statement

StudioVerse currently needs a formal Manage Users capability for **Company** and **Professional** roles so these users can onboard the users that sit within their operational scope.

Without this capability:
- Companies must rely on Super Admin for all user onboarding,
- Professionals cannot directly onboard their Individuals,
- tenant and coach associations may become manual or inconsistent,
- and user creation becomes slower than needed for real operational use.

The platform therefore needs a role-aware delegated user creation flow with clear rules for what each role can create and how the new user is associated.

---

## 6. Objectives

The objectives of this Epic are:

- Extend Manage Users to Company and Professional roles.
- Reuse the existing role model already working for Super Admin.
- Allow Company to create **Professional** and **Individual** users.
- Allow Professional to create **Individual** users.
- Automatically associate new users to the correct tenant when created by Company.
- Automatically associate a newly created Individual to the logged-in Professional when created by Professional.
- Remove tenant selection from Company and Professional creation flows where tenant is already known.
- Enforce mandatory fields for each creation type.
- Show optional coach selection only when Company creates an Individual.
- Populate coach dropdown from existing Professionals of the same Company.
- Ensure created users are visible only within the correct scope.
- Keep the user creation flow simple, fast, and operationally scalable.

---

## 7. Non-Objectives

The following are out of scope for this Epic:

- editing advanced profile data during user creation,
- bulk upload or CSV-based user import,
- user deletion workflows,
- role conversion after creation,
- user invitation email or SMS workflows,
- and multi-company reassignment logic.

This Epic focuses on scoped creation and association of users.

## 10. Domain Definitions

### 10.1 Manage Users
Manage Users is the module through which permitted roles can create and view users within their allowed scope.

### 10.2 Company User Creation
Company User Creation is the flow in which a logged-in Company user creates either a Professional or an Individual within the same tenant.

### 10.3 Professional User Creation
Professional User Creation is the flow in which a logged-in Professional creates an Individual associated with that Professional.

### 10.4 Tenant Inheritance
Tenant Inheritance is the rule that the created user automatically receives the tenant of the logged-in Company creator, without requiring tenant selection.

### 10.5 Coach Association
Coach Association is the linkage between an Individual and a Professional.

### 10.6 Scoped Visibility
Scoped Visibility is the rule that a role can only view and manage users that fall within its defined access boundary.

---

## 11. User Roles

### 11.1 Super Admin
Super Admin already has working functionality to create Company, Professional, and Individual users across the platform.

### 11.2 Company
A Company user can create:
- Professional users,
- Individual users.

A Company user cannot create another Company or Super Admin user.

### 11.3 Professional
A Professional user can create:
- Individual users only.

A Professional user cannot create Company or Professional users.

### 11.4 Individual
An Individual user cannot create users.

---

## 12. Access Scope

### 12.1 Company Scope
A Company user shall be able to manage only Professionals and Individuals that belong to the same Company / tenant.

### 12.2 Professional Scope
A Professional user shall be able to manage only Individuals associated to that Professional.

### 12.3 Super Admin Scope
Super Admin remains unchanged and continues to manage all users platform-wide.

---

## 13. Creation Rules by Role

### 13.1 Company Can Create Professional
When a Company creates a Professional:
- tenant selection shall not be shown,
- the created Professional shall automatically inherit the Company tenant,
- the form shall require first name, last name, phone number, and email address.

### 13.2 Company Can Create Individual
When a Company creates an Individual:
- tenant selection shall not be shown,
- the created Individual shall automatically inherit the Company tenant,
- the form shall require first name, last name, phone number, and email address,
- the form shall show a **Coach** dropdown,
- the Coach dropdown shall be optional,
- the Coach dropdown shall display existing Professionals of that same Company.

### 13.3 Professional Can Create Individual
When a Professional creates an Individual:
- tenant selection shall not be shown,
- coach selection shall not be shown,
- the created Individual shall automatically inherit the Professional’s tenant,
- the created Individual shall automatically be associated with the logged-in Professional,
- the form shall require first name, last name, phone number, and email address.

### 14.3 Create Individual Form for Professional

## 15. Association Logic

### 15.1 Tenant Association for Company-Created Users
Any Professional or Individual created by Company shall automatically be associated to the same tenant as the logged-in Company.

### 15.2 Coach Association for Company-Created Individuals
If the optional Coach dropdown is selected while Company creates an Individual:
- the created Individual shall be associated to the selected Professional.

If no coach is selected:
- the Individual shall still be created successfully,
- and the coach association shall remain blank / unassigned.

### 15.3 Professional Association for Professional-Created Individuals
When a Professional creates an Individual:
- the created Individual shall automatically be associated with the logged-in Professional,
- no coach selection shall be required,
- and no coach selection shall be shown.

---

## 16. Validation Rules

### 16.1 Mandatory Fields
The system shall require:
- first name,
- last name,
- phone number,
- email address

for all supported creation flows in this Epic.

### 16.2 Email Validation
The system shall validate that the email address is in a valid email format.

### 16.3 Phone Validation
The system shall validate that the phone number is in a valid format according to the product’s chosen phone validation rules.

### 16.4 Coach Dropdown Validation
For Company-created Individuals:
- coach selection shall be optional,
- if selected, the selected coach must belong to the same Company.

### 16.5 Duplicate Handling
The product should prevent duplicate user creation based on platform identity rules such as existing email address or phone number, depending on the broader identity model.

---

## 17. User Experience Requirements

### 17.1 Manage Users Page
The Manage Users page for Company and Professional shall allow the logged-in user to:
- view users in scope,
- open create-user flow,
- select the allowed target user type,
- and create users quickly without unnecessary tenant input.

### 17.2 Simplified Flow
The creation form shall be simplified based on creator role so that users do not see fields that the system already knows.

### 17.3 Role-Aware Form Rendering
The system shall render the form differently based on:
- who is logged in,
- what user type is being created.

### 17.4 Post-Creation Feedback
After successful creation, the system shall show clear confirmation and refresh the managed user listing appropriately.

## 19. Business Rules

- Super Admin user creation remains unchanged and already works across all user types.
- Company can create only Professional and Individual users.
- Professional can create only Individual users.
- Individual cannot create users.
- Tenant selection shall not be shown to Company when creating Professional or Individual.
- Tenant selection shall not be shown to Professional when creating Individual.
- Company-created users must inherit the Company tenant.
- Company-created Individual may optionally be assigned to a coach from existing Professionals of that Company.
- Professional-created Individual must automatically be assigned to the logged-in Professional.
- First name, last name, phone number, and email address are mandatory in all supported creation flows.
- Coach dropdown is shown only when Company creates an Individual.
- Coach dropdown is optional.
- Coach dropdown values must be restricted to Professionals from the same Company.

---

## 20. Architectural Recommendation

The Manage Users capability should be implemented as one reusable role-aware user creation framework rather than building separate unrelated screens for each role.

The recommended architecture is:
- one common Manage Users module,
- one shared user creation service,
- one role-based form renderer,
- one tenant association resolver,
- and one association rules engine for coach / professional linkage.

This approach improves:
- consistency,
- maintainability,
- validation reuse,
- extensibility,
- and auditability.

## 22. User Stories

### US-E10-01 — Company Can Create Professional
**As a** Company user,  
**I want** to create a Professional within my company,  
**so that** I can onboard coaches / professionals without Super Admin dependency.

**Acceptance Criteria**
- Company can select Create Professional.
- Form requires first name, last name, phone number, and email address.
- Tenant is auto-associated and not shown.
- Professional is created under the same Company / tenant.

---

### US-E10-02 — Company Can Create Individual
**As a** Company user,  
**I want** to create an Individual within my company,  
**so that** I can onboard participants directly.

**Acceptance Criteria**
- Company can select Create Individual.
- Form requires first name, last name, phone number, and email address.
- Coach dropdown is shown and optional.
- Tenant is auto-associated and not shown.
- If coach is selected, the Individual is linked to that coach.

---

### US-E10-03 — Professional Can Create Individual
**As a** Professional user,  
**I want** to create an Individual under me,  
**so that** I can onboard my own participants directly.

**Acceptance Criteria**
- Professional can select Create Individual.
- Form requires first name, last name, phone number, and email address.
- Tenant selector is not shown.
- Coach selector is not shown.
- Created Individual is automatically linked to the logged-in Professional.

---

### US-E10-04 — Role Scope Is Enforced
**As a** Product Owner,  
**I want** each role to create only allowed user types,  
**so that** delegated user creation stays controlled.

**Acceptance Criteria**
- Company cannot create Company users.
- Company cannot create Super Admin users.
- Professional cannot create Professional users.
- Professional cannot create Company users.
- Individual cannot create any users.

---

### US-E10-05 — Coach Dropdown Is Restricted Correctly
**As a** Company user,  
**I want** to see only relevant coaches when assigning an Individual,  
**so that** I do not link users across the wrong company.

**Acceptance Criteria**
- Coach dropdown shows only Professionals from the same Company.
- Dropdown is optional.
- No cross-company Professionals are shown.

- Company cannot create Company,

- Company-created Individual is associated with the same tenant,

- coach dropdown values belong only to the same Company,
- Professional-created Individual is automatically linked to logged-in Professional,

---

## 24. Risks and Considerations

- If tenant inheritance is not enforced correctly, users may be created in the wrong tenant.
- If coach dropdown is not filtered correctly, cross-company association errors may occur.
- If Professional-created Individuals are not auto-linked reliably, downstream cohort and activity behavior may fail.
- If duplicate identity rules are weak, user duplication may increase.
- If role-based restrictions are not enforced at backend level, unauthorized creation paths may appear.
